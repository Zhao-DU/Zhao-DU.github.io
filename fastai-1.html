Here’s a set of notes for FastAI, which is high-level API of pytorch. This post focuses on the fastai.vision API.

Note for vision API

General tips

  • You can use from fastai.vision.all import * although it is generally discouraged for python, because the library has been specifically designed for the purpose, i.e.

    
from fastai.vision.all import *
from fastai.vision.widgets import *

  • When in doubt, call doc() on any of the fastAI functions to see documentation.

Creating Path Object

path can basically be used instead of the typical os.path or magic shell commands.

  • path.ls(file_exts='.pkl') will list all pickled models
  • use Path.BASE_PATH to set root directory for your project, thus allowing for dataset definition/manipulation using relative path
  • You can directly use / to access relative directory. i.e. path/'train'

path = untar_data(URLs.MNIST_SAMPLE)
Path.BASE_PATH = path
print(Path.BASE_PATH) # outputs /root/.fastai/data/mnist_sample. Note, using print() and simply inputing Path.BASE_PATH into jupyter backend yields different results
(path/'train').ls() # Use the integrated ls function to list

Helper functions for working with path:

  • get_images_files(path): get all images recursively in a path. Returns fastcore.foundation.L instance. Should work for all standard image extensions.

A complete list for helper functions is found here

Some useful functions for loading can be accessed as methods of the ImageDataLoaders class

Datablock API

Data augmentation: item_tfms

Image augmentation/transforms for fastAI is GPU-accelerated. It is most easily accessed through the aug_transforms function.

By default, augmentation will not occur on the validation set, with the exception of RandomResizedCrop

Factory Dataloaders

from_path_func: Create from list of fnames which exist in path


path = untar_data(URLs.MNIST_TINY)
fnames = get_image_files(path)
def label_func(x): return x.parent.name
dls = ImageDataLoaders.from_path_func(path, fnames, label_func)

Building a cNN with pre-defined architecture


from fastai.vision.all import *

learn = cnn_learner(dls=dataloader, arch=resnet34, metrics=error_rate)

Complete list of parameters for cnn_learner

Data Cleaning

After training

While it may sound counter-intuitive to clean the data after you’ve trained the model, you can use ImageClassifierCleaner(*model*) to have the model output the examples with largest loss.

There is then a nice GUI within jupyter to select samples to remove/re-label. (Note when you use the drop-down menu, it doesn’t actually delete/re-label the sample, but only saves its index. You need to then call unlink and shutil.move as seen below.)

This should help you rapidly check for the examples which are more ambiguous, although at the cost that you might have to re-train your model after removing the problematic samples. 


# Bring up the output of the 
cleaner = ImageClassifierCleaner(learn)
cleaner

# Remove links for images labeled as "delete"
for idx in cleaner.delete(): cleaner.fns[idx].unlink()

# Move samples with changed label into correct directory 
for idx,cat in cleaner.change(): shutil.move(str(cleaner.fns[idx]), path/cat)

Inference

learner.predict(): predict probability.
learner.dls.vocab: see list of labels for list of probability that model outputs in inference.
